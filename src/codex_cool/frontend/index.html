使用文档

Codex App

在仪表盘点击「注入代理」按钮，自动写入 ~/.codex/config.toml

启动 Codex App，在模型选择器中选择 Codex-Cool 提供的模型即可。如需切回原始配置，点击「恢复配置」。

支持的 API 格式：Responses（推荐）、Chat Completions

检查点功能：通过 previous_response_id 实现链式对话上下文传递

Codex CLI

编辑 ~/.codex/config.toml：

model = "deepseek-chat" model_provider = "codex-cool" [model_providers."codex-cool"] name = "Codex-Cool" base_url = "http://127.0.0.1:18080/v1" wire_api = "responses"

配置完成后直接在终端使用 codex 命令即可

Claude App

在仪表盘点击「注入代理」按钮，自动配置 Claude App 第三方推理网关

注入后重启 Claude App 即可使用第三方 API，无需登录。如需切回官方登录，点击「恢复配置」。

⚠️ 注入前请确保 Claude Desktop 已关闭，否则可能写入失败

注入自动完成：切换 3P 模式 → 开启 Developer Mode → 设置 NODE_ENV=production（macOS）

支持多模型注入：可选择多个模型映射到 Claude 的 Opus/Sonnet/Haiku 等级

Claude Code

设置环境变量后启动：

export ANTHROPIC_BASE_URL=http://127.0.0.1:18080 claude

或在 ~/.claude/settings.json 中添加：

{"apiBaseUrl": "http://127.0.0.1:18080"}

Claude Code 使用 Anthropic Messages 协议，自动转换为上游 Provider 格式

环境变量

写入 shell 配置（如 ~/.zshrc 或 ~/.bashrc）：

export CODEX_COOL_API_KEY="your-api-key"

之后从 Dock 启动 Codex App 也能读到 Key

也可在供应商配置中使用 env:VAR_NAME 引用环境变量，避免明文写入配置文件

API 代理

任何兼容 OpenAI / Anthropic 的第三方客户端均可连接：

Base URL: http://127.0.0.1:18080/v1 API Key: （任意值或留空）

支持协议：OpenAI Responses、OpenAI Chat Completions、Anthropic Messages

协议自动转换：无论客户端发什么格式，都会自动转换成上游 Provider 需要的格式

多模态处理

以下 Provider 不支持图片/文件输入，发送时自动转换为占位符：

DeepSeek Kimi 智谱 千问 豆包

首次选择此类模型时会弹出提醒，后续不再重复提示。图片会被替换为 [image] 占位符文本。
